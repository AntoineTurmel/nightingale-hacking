Detailed Description

Metadata Job represents the status of a running metadata scanning task.

You get an sbIMetadataJob by calling sbIMetadataJobManager::newJob() with an nsIArray of sbIMediaItem.

See also:
sbILibrary, sbIMediaItem, sbIMetadataJobManager

void sbIMetadataJob::init ( in AString  aTableName,
in nsIArray  aMediaItemsArray,
in unsigned long  aSleepMS 
)

Initialize a task with a specific guid and, optionally, an nsIArray of sbMediaItems.

The metadata task will track its progress through a database table so it can depend on its information always available upon app restart.

There is a window of time while it is copying the given array into the database that a crash or shutdown will lose the enqueueing of the task.

Parameters:
aTableName The table name assigned to this task.
aMediaItemsArray An array filled with media items to be scanned.
aSleepMS How much of an interval/sleep per loop in and out of thread.

void sbIMetadataJob::append ( in nsIArray  aMediaItemsArray  ) 

Add some more media items to the job's queue.

Appending to a completed job will throw an error.

THIS METHOD APPENDS ALL THE ITEMS TO THE DATABASE QUEUE, BLOCKING UNTIL COMPLETE. Try to only give it small (< 500 items) arrays in batches.

Parameters:
aMediaItemsArray An array filled with media items to be scanned.
